## Avatar VR con movimiento mediante ROS2 topics

Estas dos semanas he estado principalmente avanzando con el avatar VR. Para el movimiento el problema principal era poder moverse según el giro de la cabeza y respetando las físicas.

Tenía dos problemas principales que nos evitaban cumplir este objetivo:

- El problema principal es que no podía usar un sistema dinámico de movimiento de velocidad para poder mover al personaje y que respete las físicas ya que el movimiento de la cabeza no se respetaría, debido a que a la hora de moverse solo se centraría en las físicas y no en que el personaje cambie su transform.

- Por ello tenía que usar el sistema de movimiento kinematic pero que no respeta las colisiones con paredes, es decir, si afecta las colisiones al pasar por delante de otras entidades movibles pero no con una pared fija.

Al final, para poder conseguir lo que ya tenía con kinematic y poder lograr colision con paredes, añadi un sistema de raycasting con una clase que también ofrece O3DE en su [API](https://www.docs.o3de.org/docs/api/frameworks/azframework/class_physics_1_1_rigid_body_requests.html). De esta manera y haciendo una pequeña pausa por cada paso que de, obtener un movimiento fluido con el giro de la cabeza y que chocara con paredes.

Código para movimiento del avatar(teniendo en cuenta que el programa python externo ROS envía un Header):

- **AvatarControl.h**
  ```cpp
  #pragma once

  #include <AzCore/std/containers/vector.h>
  #include <AzCore/Component/Component.h>
  #include <AzCore/Component/EntityId.h>
  #include <AzCore/Component/TickBus.h>

  #include <ROS2/ROS2Bus.h>
  #include <ROS2/Utilities/ROS2Names.h>

  #include <rclcpp/subscription.hpp>
  #include <std_msgs/msg/header.hpp>

  class AvatarControl
    : public AZ::Component
    , public AZ::TickBus::Handler
  {
  public:
      AZ_COMPONENT(AvatarControl,"{ed3cf58d-5c14-4bc4-9313-b35db0d8526f}");
      AvatarControl();
      ~AvatarControl()=default;

      static void Reflect(AZ::ReflectContext* context);
      void OnTick(float deltaTime, AZ::ScriptTimePoint time) override;

      void Activate() override;
      void Deactivate() override;

  private:
      void TopicCallBack(const std_msgs::msg::Header::SharedPtr);
      void MoveEntity(AZ::EntityId entityMove, const std_msgs::msg::Header::SharedPtr msg);

      std::shared_ptr<rclcpp::Subscription<std_msgs::msg::Header>> m_subscription;
      AZ::EntityId moveEntity;
      float speed;
      bool m_isMoving = false;
      float m_waitTimer = 0.0f;
      float m_waitDuration = 0.2f;
      ROS2::TopicConfiguration m_avatarControlTopicConfiguration;
  };
  ```

- **AvatarControl.cpp**
  ```cpp
  #include "AvatarControl.h"
  #include <AzCore/Serialization/SerializeContext.h>
  #include <AzCore/Math/Vector3.h>
  #include <ROS2/ROS2Bus.h>
  #include <AzFramework/Physics/RigidBodyBus.h>
  #include <AzCore/Component/TickBus.h>
  #include <AzFramework/Physics/PhysicsScene.h>
  #include <AzFramework/Physics/Common/PhysicsSceneQueries.h>
  #include <AzCore/Interface/Interface.h>

  AvatarControl::AvatarControl(){
      m_avatarControlTopicConfiguration.m_topic="/avatar_control";
      m_avatarControlTopicConfiguration.m_type="std_msgs::msg::Header";
  }

  void AvatarControl::Reflect(AZ::ReflectContext* context){
      if(auto serializeContext = azrtti_cast<AZ::SerializeContext*>(context)){
          serializeContext->Class<AvatarControl, AZ::Component>()
          ->Version(1)
          ->Field("Move Entity", &AvatarControl::moveEntity)
          ->Field("Speed", &AvatarControl::speed)
          ->Field("Avatar Control Topic", &AvatarControl::m_avatarControlTopicConfiguration);

          if (auto editContext = serializeContext->GetEditContext())
          {
              editContext->Class<AvatarControl>("AvatarControl", "Control for avatar")
                  ->ClassElement(AZ::Edit::ClassElements::EditorData, "")
                  ->Attribute(AZ::Edit::Attributes::Category, "Custom")
                  ->Attribute(AZ::Edit::Attributes::AppearsInAddComponentMenu, AZ_CRC("Game"))
                  ->DataElement(AZ::Edit::UIHandlers::Default, &AvatarControl::moveEntity,"Move Entity","Entity to move")
                  ->DataElement(AZ::Edit::UIHandlers::Default, &AvatarControl::speed, "Move speed", "Speed movement")
                  ->DataElement(
                          AZ::Edit::UIHandlers::Default,
                          &AvatarControl::m_avatarControlTopicConfiguration,
                          "Topic for avatar control message",
                          "Configuration for ROS2 topic to recieve avatar control message");
          }
      }
  }

  void AvatarControl::OnTick(float deltaTime, AZ::ScriptTimePoint)
  {
      if(m_isMoving){
          m_waitTimer += deltaTime;

          if (m_waitTimer >= m_waitDuration)
          {
              m_waitTimer = 0.0f;
              m_isMoving = false;
          }
      }
  }

  void AvatarControl::Activate(){
      AZ::TickBus::Handler::BusConnect();
      auto m_node = ROS2::ROS2Interface::Get()->GetNode();
      m_subscription = m_node->create_subscription<std_msgs::msg::Header>(m_avatarControlTopicConfiguration.m_topic.data(), m_avatarControlTopicConfiguration.GetQoS(), std::bind(&AvatarControl::TopicCallBack, this, std::placeholders::_1));
  }

  void AvatarControl::Deactivate(){
      AZ::TickBus::Handler::BusDisconnect();
      m_subscription.reset();
  }

  void AvatarControl::TopicCallBack(const std_msgs::msg::Header::SharedPtr msg){
      if(!moveEntity.IsValid())
      {
          AZ_Warning("AvatarControl", false, "Unknow Move Entity");
          return;
      }

      MoveEntity(moveEntity, msg);
  }

  void AvatarControl::MoveEntity(AZ::EntityId entityMove, const std_msgs::msg::Header::SharedPtr msg)
  {

      if (m_isMoving) return;

      AZ::Vector3 localDirection;
      AZ::Transform currentTransform;

      char comand = msg->frame_id[0];

      switch(comand){
          case 'w': localDirection=AZ::Vector3(0, speed, 0); break;
          case 'a': localDirection=AZ::Vector3(-speed, 0, 0); break;
          case 's': localDirection=AZ::Vector3(0, -speed, 0); break;
          case 'd': localDirection=AZ::Vector3(speed, 0, 0); break;
      }

      AZ::TransformBus::EventResult(currentTransform, entityMove, &AZ::TransformBus::Events::GetWorldTM);

      AZ::Vector3 currentPosition = currentTransform.GetTranslation();
      AZ::Vector3 forward = currentTransform.GetBasisY();
      forward.SetZ(0.0f);

      if (forward.GetLengthSq() < 0.001f)
      {
          forward = AZ::Vector3(0.0f, 1.0f, 0.0f);
      }
      else
      {
          forward.Normalize();
      }

      AZ::Vector3 right = currentTransform.GetBasisX();
      right.SetZ(0.0f);
      
      if (right.GetLengthSq() < 0.001f)
      {
          right = AZ::Vector3(1.0f, 0.0f, 0.0f);
      }
      else
      {
          right.Normalize();
      }

      AZ::Vector3 globalDirection = right   * localDirection.GetX() + forward * localDirection.GetY();
      globalDirection.SetZ(0.0f);

      AZ::Vector3 startPos = currentPosition;
      AZ::Vector3 finalPos = startPos+globalDirection;

      auto* sceneInterface = AZ::Interface<AzPhysics::SceneInterface>::Get();

      AzPhysics::RayCastRequest request;
      request.m_start = startPos;
      request.m_direction = globalDirection.GetNormalized();
      request.m_distance = globalDirection.GetLength();
      request.m_filterCallback = [entityMove](const AzPhysics::SimulatedBody* body, const Physics::Shape* shape)
      {
          if (body && body->GetEntityId() == entityMove)
              return AzPhysics::SceneQuery::QueryHitType::None;
          return AzPhysics::SceneQuery::QueryHitType::Touch;
      };

      AzPhysics::SceneHandle sceneHandle = sceneInterface->GetSceneHandle(AzPhysics::DefaultPhysicsSceneName);
      AzPhysics::SceneQueryHits result = sceneInterface->QueryScene(sceneHandle, &request);

      if (!result.m_hits.empty()){
          const auto& hit = result.m_hits[0];
          AZ_Warning("AvatarControl", false, "Hit normal: %f %f %f, distance: %f", hit.m_normal.GetX(), hit.m_normal.GetY(), hit.m_normal.GetZ(), hit.m_distance);
          float safeDistance = AZStd::max(0.0f, hit.m_distance - 0.1f);
          finalPos = startPos + request.m_direction * safeDistance;
      }

      currentTransform.SetTranslation(finalPos);

      Physics::RigidBodyRequestBus::Event(entityMove, &Physics::RigidBodyRequests::SetKinematicTarget, currentTransform);

      m_isMoving=true;
  }
  ```

## Avances configuración brazo robótico en O3DE

Tmabién he estado trabajando, en un segundo plano, con la implementación del brazo robótico ur5.

Ya logramos importarlo pero ahora había que cambiar y añadir varios componentes de brazo robótico, que se encuentran mencionado en la documentación de O3DE en el apartado de [Joints Manipualtion](https://www.docs.o3de.org/docs/user-guide/interactivity/robotics/joints-manipulation/) y [Grippers](https://www.docs.o3de.org/docs/user-guide/interactivity/robotics/grippers/).

El problema y con lo que aún estoy trabajando es con la configuración de estos componentes, que para saber cómo configurarlos necesito entender como está configurado el ur5. Para ello, me estoy comunicando con un compañero de la empresa que trabaja con brazos robóticos para poder entender la configuración del brazo e implementarlo en O3DE.

Por ahora tengo una idea del controlador de joints y el server del gripper, ya que esto lo he podido ver en la carpeta config del paquete ros del brazo robótico en **controllers.yaml**:
