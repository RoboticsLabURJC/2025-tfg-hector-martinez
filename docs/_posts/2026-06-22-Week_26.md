## Uso de ROS para sincronizar cliente-host en sistema VR en FollowPerson

Durante este mes he estado trabajando en la implementación del sistema VR en el RADIx. La idea principal era tener una escena del robot en tercera persona y por otro lado la escena en los cascos de realidad virtual en primera persona desde el POV de la persona.

La única manera para conseguir que dos Game Camera para el mismo level es mediante el uso de la [Gem Multiplayer](https://docs.o3de.org/docs/user-guide/networking/multiplayer/overview/), con la instancia host (escena tercera persona de turtlebot) y la instancia cliente (escena POV de la persona) que se conecta a la instancia host.

De esta manera la estructura dentro del RADIx a la hora de lanzar el ejercicio FollowPerson sería la siguiente:

![Imagen Arquitectura Sistema VR en RADIx](Architecture_VR_System_RADIx.png).

Un problema con este método es que cada instancia es independiente, es decir, si el robot se mueve en la instancia host, en la instancia cliente no se mueve. Para arreglar esto en principio se había tenido en cuenta [auto-components](https://docs.o3de.org/docs/user-guide/networking/multiplayer/autocomponents/) como se menciono en el último post, pero estos eran demasiado complejos para una mécanica tan sencilla como copiar posición y rotación. Luego se optó por componentes base de la Gem Multiplayer que se encargan de justo copiar el transform del host en el cliente, pero que no funcionaban correctamente.

Al final opté por usar ROS topics y crear dos tipos de comunicación maestro-esclavo:

- **Maestro Host-Esclavo Cliente**: El host envía el transform del robot y del cuerpo de la persona al cliente y el cliente recibe y escribe ese transform.
- **Maestro Cliente-Esclavo Host**: El cliente envía el transform de la cabeza de la persona al host y el host recibe y escribe esa rotación. Este se hace para que la persona pueda mover la cabeza desde la instancia cliente.

De esta manera queda el código CameraManager cambiado el nombre a ManagerVR para mejor compresión:

**ManagerVR.h**
```cpp
#pragma once
#include <AzCore/Component/Component.h>
#include <AzCore/Component/EntityId.h>
#include <AzCore/Component/TickBus.h>
#include <AzCore/Math/Transform.h>
#include <rclcpp/rclcpp.hpp>
#include <geometry_msgs/msg/transform_stamped.hpp>
#include <geometry_msgs/msg/quaternion.hpp>

class ManagerVR
    : public AZ::Component
    , public AZ::TickBus::Handler
{
public:
    AZ_COMPONENT(ManagerVR, "{e4492198-576b-4d18-996a-f0e9f4ffc199}");
    ManagerVR();
    ~ManagerVR() = default;
    static void Reflect(AZ::ReflectContext* context);
    void OnTick(float deltaTime, AZ::ScriptTimePoint time) override;
    void Activate() override;
    void Deactivate() override;

private:
    void ManageCameras();
    bool IsHost();
    geometry_msgs::msg::TransformStamped BuildTransformMsg(const AZ::Transform& tm);
    void PublishTransforms();
    void ApplyHeadRotation();
    void ApplyTransforms();

    // Callbacks
    void OnBodyTransformReceived(const geometry_msgs::msg::TransformStamped::SharedPtr msg);
    void OnRobotTransformReceived(const geometry_msgs::msg::TransformStamped::SharedPtr msg);
    void OnHeadRotationReceived  (const geometry_msgs::msg::TransformStamped::SharedPtr msg);

    AZ::EntityId cameraClient;
    AZ::EntityId cameraHost;
    AZ::EntityId bodyEntity;  
    AZ::EntityId headEntity;
    AZ::EntityId robotEntity;

    // ROS2
    rclcpp::Node::SharedPtr m_ros2Node;

    rclcpp::Subscription<geometry_msgs::msg::TransformStamped>::SharedPtr m_bodySub;
    rclcpp::Subscription<geometry_msgs::msg::TransformStamped>::SharedPtr m_headRotSub;
    rclcpp::Subscription<geometry_msgs::msg::TransformStamped>::SharedPtr  m_robotSub;

    rclcpp::Publisher<geometry_msgs::msg::TransformStamped>::SharedPtr m_bodyPub;
    rclcpp::Publisher<geometry_msgs::msg::TransformStamped>::SharedPtr m_headRotPub;
    rclcpp::Publisher<geometry_msgs::msg::TransformStamped>::SharedPtr m_robotPub;

    AZ::Transform m_receivedBodyTransform   = AZ::Transform::CreateIdentity();
    AZ::Quaternion m_receivedHeadRotation   = AZ::Quaternion::CreateIdentity();
    AZ::Transform  m_receivedRobotTransform  = AZ::Transform::CreateIdentity();
    bool m_hasReceivedHeadRotation          = false;
    bool m_robotKinematicSet = false;
};
```

**ManagerVR.cpp**
```cpp
#include "ManagerVR.h"
#include <AzCore/Serialization/SerializeContext.h>
#include <AzCore/Serialization/EditContext.h>
#include <AzCore/Component/TransformBus.h>
#include <AzCore/Interface/Interface.h>
#include <AzFramework/Components/CameraBus.h>
#include <Multiplayer/IMultiplayer.h>

ManagerVR::ManagerVR() {}

void ManagerVR::Reflect(AZ::ReflectContext* context)
{
    if (auto serializeContext = azrtti_cast<AZ::SerializeContext*>(context))
    {
        serializeContext->Class<ManagerVR, AZ::Component>()
            ->Version(1)                                          
            ->Field("Avatar Entity",  &ManagerVR::cameraClient)
            ->Field("Robot Entity",   &ManagerVR::cameraHost)
            ->Field("Robot Entity3D", &ManagerVR::robotEntity)
            ->Field("Body Entity",    &ManagerVR::bodyEntity)   
            ->Field("Head Entity",    &ManagerVR::headEntity);  

        if (auto editContext = serializeContext->GetEditContext())
        {
            editContext->Class<ManagerVR>("ManagerVR", "Manage System VR")
                ->ClassElement(AZ::Edit::ClassElements::EditorData, "")
                    ->Attribute(AZ::Edit::Attributes::Category, "Custom")
                    ->Attribute(AZ::Edit::Attributes::AppearsInAddComponentMenu, AZ_CRC("Game"))
                ->DataElement(AZ::Edit::UIHandlers::Default,
                    &ManagerVR::cameraClient, "Camara VR", "")
                ->DataElement(AZ::Edit::UIHandlers::Default,
                    &ManagerVR::cameraHost, "Camara Host", "")
                ->DataElement(AZ::Edit::UIHandlers::Default,
                    &ManagerVR::robotEntity, "Robot", "")
                ->DataElement(AZ::Edit::UIHandlers::Default,
                    &ManagerVR::bodyEntity, "Body", "")        
                ->DataElement(AZ::Edit::UIHandlers::Default,
                    &ManagerVR::headEntity, "Head", "");       
        }
    }
}

void ManagerVR::Activate()
{
    m_ros2Node = std::make_shared<rclcpp::Node>("camera_manager");

    if (IsHost())
    {
        m_bodyPub = m_ros2Node->create_publisher<geometry_msgs::msg::TransformStamped>(
            "/sync/body_transform", 10);
        m_robotPub = m_ros2Node->create_publisher<geometry_msgs::msg::TransformStamped>(
            "/sync/robot_transform", 10);
        m_headRotSub = m_ros2Node->create_subscription<geometry_msgs::msg::TransformStamped>(
            "/sync/head_rotation", 10,
            std::bind(&ManagerVR::OnHeadRotationReceived, this, std::placeholders::_1));
    }
    else
    {
        m_bodySub = m_ros2Node->create_subscription<geometry_msgs::msg::TransformStamped>(
            "/sync/body_transform", 10,
            std::bind(&ManagerVR::OnBodyTransformReceived, this, std::placeholders::_1));

        m_robotSub = m_ros2Node->create_subscription<geometry_msgs::msg::TransformStamped>(
            "/sync/robot_transform", 10,
            std::bind(&ManagerVR::OnRobotTransformReceived, this, std::placeholders::_1));

        m_headRotPub = m_ros2Node->create_publisher<geometry_msgs::msg::TransformStamped>(
            "/sync/head_rotation", 10);
    }

    AZ::TickBus::Handler::BusConnect();
}

void ManagerVR::Deactivate()
{
    AZ::TickBus::Handler::BusDisconnect();
    m_bodyPub.reset();
    m_robotPub.reset();
    m_bodySub.reset();
    m_robotSub.reset();
    m_headRotPub.reset();
    m_headRotSub.reset();
    m_ros2Node.reset();
}

void ManagerVR::OnBodyTransformReceived(
    const geometry_msgs::msg::TransformStamped::SharedPtr msg)
{
    auto& t = msg->transform.translation;
    auto& r = msg->transform.rotation;
    m_receivedBodyTransform = AZ::Transform::CreateFromQuaternionAndTranslation(
        AZ::Quaternion(r.x, r.y, r.z, r.w),
        AZ::Vector3(t.x, t.y, t.z));
}

void ManagerVR::OnRobotTransformReceived(
    const geometry_msgs::msg::TransformStamped::SharedPtr msg)
{
    auto& t = msg->transform.translation;
    auto& r = msg->transform.rotation;
    m_receivedRobotTransform = AZ::Transform::CreateFromQuaternionAndTranslation(
        AZ::Quaternion(r.x, r.y, r.z, r.w),
        AZ::Vector3(t.x, t.y, t.z));
}

void ManagerVR::OnHeadRotationReceived(
    const geometry_msgs::msg::TransformStamped::SharedPtr msg)
{
    auto& r = msg->transform.rotation;
    m_receivedHeadRotation = AZ::Quaternion(r.x, r.y, r.z, r.w);
    m_hasReceivedHeadRotation = true;
}

bool ManagerVR::IsHost()
{
    auto* multiplayer = AZ::Interface<Multiplayer::IMultiplayer>::Get();
    if (!multiplayer) return false;
    return (multiplayer->GetAgentType() == Multiplayer::MultiplayerAgentType::ClientServer);
}

void ManagerVR::ManageCameras()
{
    if (IsHost())
        Camera::CameraRequestBus::Event(cameraHost, &Camera::CameraRequestBus::Events::MakeActiveView);
    else
        Camera::CameraRequestBus::Event(cameraClient, &Camera::CameraRequestBus::Events::MakeActiveView);
}

geometry_msgs::msg::TransformStamped ManagerVR::BuildTransformMsg(const AZ::Transform& tm)
{
    geometry_msgs::msg::TransformStamped msg;
    auto t = tm.GetTranslation();
    auto r = tm.GetRotation();
    msg.transform.translation.x = t.GetX();
    msg.transform.translation.y = t.GetY();
    msg.transform.translation.z = t.GetZ();
    msg.transform.rotation.x = r.GetX();
    msg.transform.rotation.y = r.GetY();
    msg.transform.rotation.z = r.GetZ();
    msg.transform.rotation.w = r.GetW();
    return msg;
}

void ManagerVR::PublishTransforms()
{
    AZ::Transform bodyTm  = AZ::Transform::CreateIdentity();
    AZ::Transform robotTm = AZ::Transform::CreateIdentity();

    AZ::TransformBus::EventResult(bodyTm,  bodyEntity,  &AZ::TransformBus::Events::GetWorldTM);
    AZ::TransformBus::EventResult(robotTm, robotEntity, &AZ::TransformBus::Events::GetWorldTM);

    m_bodyPub->publish(BuildTransformMsg(bodyTm));
    m_robotPub->publish(BuildTransformMsg(robotTm));
}

void ManagerVR::ApplyHeadRotation()
{
    if (!m_hasReceivedHeadRotation) return;
    if (!headEntity.IsValid()) return;

    AZ::TransformBus::Event(headEntity,
        &AZ::TransformBus::Events::SetWorldRotationQuaternion,
        m_receivedHeadRotation);
}

void ManagerVR::ApplyTransforms()
{
    rclcpp::spin_some(m_ros2Node);

    if (bodyEntity.IsValid())
        AZ::TransformBus::Event(bodyEntity, &AZ::TransformBus::Events::SetWorldTM,
            m_receivedBodyTransform);

    if (robotEntity.IsValid())
        AZ::TransformBus::Event(robotEntity, &AZ::TransformBus::Events::SetWorldTM,
            m_receivedRobotTransform);

    if (headEntity.IsValid() && m_headRotPub)
    {
        AZ::Transform headTm = AZ::Transform::CreateIdentity();
        AZ::TransformBus::EventResult(headTm, headEntity,
            &AZ::TransformBus::Events::GetWorldTM);

        geometry_msgs::msg::TransformStamped msg;
        auto r = headTm.GetRotation();
        msg.transform.rotation.x = r.GetX();
        msg.transform.rotation.y = r.GetY();
        msg.transform.rotation.z = r.GetZ();
        msg.transform.rotation.w = r.GetW();
        m_headRotPub->publish(msg);
    }
}

void ManagerVR::OnTick(float deltaTime, AZ::ScriptTimePoint)
{
    auto* multiplayer = AZ::Interface<Multiplayer::IMultiplayer>::Get();
    if (!multiplayer) return;
    if (multiplayer->GetAgentType() == Multiplayer::MultiplayerAgentType::Uninitialized) return;

    ManageCameras();

    if (IsHost())
    {
        rclcpp::spin_some(m_ros2Node);
        PublishTransforms();
        ApplyHeadRotation();
    }
    else
    {
        if (!m_robotKinematicSet && robotEntity.IsValid()){
            Physics::RigidBodyRequestBus::Event(robotEntity, &Physics::RigidBodyRequests::SetKinematic, true);
            m_robotKinematicSet = true;
        }
        ApplyTransforms();
    }
}
```

## Conexión SteamVR en contenedor docker

Para saber como avanzar mire un poco como funciona tanto el sistema de SteamVR mediante la [wiki de software de Valve](https://deepwiki.com/ValveSoftware/Proton/5.1-openvr-support-(vrclient_x64)) como también el funcionamiento de O3DE a la hora de detectar e iniciar el entorno virtual en las gafas, que lo hace mediante la Gem OpenXRVk, en [OpenXRVkInstance.cpp](https://github.com/o3de/o3de-extras/blob/2310.3/Gems/OpenXRVk/Code/Source/OpenXRVkInstance.cpp) que el código que se encarga de todo el tema relacionado a la instancia que crea O3DE para las gafas VR.

La idea principal sería lograr una comunicación de SteamVR en la máquina host y O3DE en un contenedor docker, para ello estoy viendo en docker mecánicas que permitan una conexión así para permitir la comunicación entre el contenedor docker y la máquina host.
